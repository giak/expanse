## 🎨 PRÉSENTATION = RESPONSABILITÉ DU FRONTEND

TOUS les aspects visuels sont calculés DANS LE FRONTEND, pas dans le JSON:
- **Couleurs noeuds**: mappées depuis `type` (APEX→rouge, REGLE→orange, etc.)
- **Couleurs liens**: mappées depuis `type` d'arête
- **Taille noeuds**: dérivée de `centrality` → `rayon = 8 + centrality * 2`
- **Positions**: calculées par le layout (couches horizontales par type)
- **Épaisseur liens**: dérivée de `weight`
- **Nature visuelle**: `nature` (permanent→solid, vivide→pulse, volatile→fade, incandescent→glow)
- **Organisation**: `parent_organ` permet le groupement par organe cognitif

Le graphe est une source de DONNÉES FONCTIONNELLES. Pas un fichier de rendu.

1. ✅ **Format compact**: `JSON.stringify(..., null, 2)` — Pretty print pour lisibilité.
2. ✅ **Dédoublonnage systématique**: Un seul nœud par ID. Une seule arête par (source, target, type).
3. ✅ **Pas d'inférence**: Aucune relation ajoutée qui n'est pas explicitement prouvée.
4. ✅ **Zéro modification**: Ce qui vient de Mnemolite reste tel quel.
5. ✅ **Validation avant écriture**: Tous les liens référencent des nœuds existants. Pas de cycle.
