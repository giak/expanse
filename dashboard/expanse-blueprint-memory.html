Ⅱ. API Mnemolite — Outils MCP

Configuration serveur : streamable-http://localhost:8002/mcp — Défini dans .kilocode/mcp.json. 8 outils alwaysAllow (pas de confirmation utilisateur requise). + 1 outil référencé par V16 mais pas dans alwaysAllow.

📖 LECTURE (3 outils alwaysAllow)

search_memory
query : recherche sémantique
tags : filtrage par tags (tableau)
limit : max résultats
sort : tri (ex: "outcome_score DESC")
consumed : bool — filtrer par statut consommation
Retourne : id, title, content, tags, outcome_score, memory_type
get_system_snapshot
repository : nom du projet (ex: "expanse")
Retourne : état complet du système Mnemolite
search_code
Recherche dans le code indexé (Vessel)
Utilisé par Φ Vessel Guard (V16 §Ⅱ)

✍️ ÉCRITURE (1 outil alwaysAllow)

write_memory
title : titre de la mémoire
content : contenu textuel
tags : tableau de tags (détermine la catégorie)
memory_type : type structuré

6 memory_type valides TESTÉ

note Observation simple
decision Décision prise (proposals, mutations)
task Tâche à faire
reference Référence durable (patterns, axiomes, diffs)
conversation Interaction archivée
investigation Analyse en cours (BRM, drifts)
⚠️ Tout autre type = ERREUR. Ex: "pattern" → INVALID

🔄 CONSOMMATION (1 outil alwaysAllow + 1 référencé)

mark_consumed alwaysAllow
memory_ids : tableau d'IDs à marquer
consumed_by : identifiant du consommateur (ex: "dream_passe1")
Effet : la mémoire ne sera plus retournée par consumed: false
rate_memory V16 §Ⅳ — PAS dans alwaysAllow
id : ID de la mémoire
helpful : True (positif) ou False (négatif)
Effet positif : module le taux de décroissance → reste plus longtemps pertinent
Effet négatif : accélère la décroissance → disparaît plus vite
Alimente outcome_score (utilisé pour le tri L2)
⚠️ V16 §Ⅳ Outcome Feedback utilise cet outil, mais .kilocode/mcp.json ne le liste pas dans alwaysAllow. Peut nécessiter confirmation utilisateur ou config update.

Chaque mémoire suit un parcours déterminé par les signaux qu'elle reçoit. Le système est auto-régulé : nul besoin d'intervention humaine sauf pour les promotions majeures.

🔥 Étincelle (auto)

Détection post-Ω silencieuse :
Q1: "M'oppose-je à un anchor?" → OUI → sys:drift
Q2: "Nouveau pattern?" → OUI → sys:pattern:candidate

⟳ Candidate → Pattern

GATE: 3 occurrences
✅ 3 interactions distinctes
✅ 3 validations indépendantes
✅ Aucun signal négatif entre les 3
ALORS: /seal → sys:pattern

★ Pattern → Core

DÉCRET USER UNIQUEMENT
/core {titre}
Migre vers sys:core + sys:anchor
= IMMORTEL, jamais consommé

→ sys:pattern:doubt
Dream Passe 4 :
SI > 3 signaux négatifs → soft-delete
SINON → réévaluer → remettre en candidate

→ soft_delete automatique
→ Ψ [ELAGUAGE] Pattern supprimé

Négatif = décroissance accélérée (disparaît plus vite)
