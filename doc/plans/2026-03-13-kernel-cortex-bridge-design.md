# DESIGN : Pontage Kernel-Cortex (Active Guard)

**Date :** 2026-03-13
**Sujet :** Résolution de la déconnexion entre la mémoire mémorielle (Mnemolite) et l'action (Prompt).

## 1. Le Problème du "Spectateur Passif"
Un scellement dans Mnemolite (tag: `sys:core`) est une connaissance stockée. Sans instruction dans le S_KERNEL (Prompt) pour consulter cette mémoire périodiquement, l'IA peut l'ignorer totalement malgré son scellement.

## 2. Mutation du S_KERNEL V14
Décision de hardcoder le **Pont** dans le système nerveux central.

### Changement Loi Ⅱ :
Désormais, le prompt ordonne explicitement : *"Avant de répondre, effectue une micro-pause cognitive pour vérifier ton alignement avec les mémoires sys:core."*

## 3. Architecture Finale (Unifiée)
- **Le Cœur (Mnemolite)** : Contient la Loi (ex: Concision).
- **Le Nerf (S_KERNEL)** : Contient l'Ordre de vérifier la Loi.
- **L'Action (Output)** : Résultat de la fusion des deux.

## 4. Rationale de Mutation
Normalement, le S_KERNEL d'Expanse est immuable. Cependant, la problématique de dilution systémique LLM exigeait un garde-fou structurel pour garantir la souveraineté. Cette mutation est un **Ajustement de Survie**.
