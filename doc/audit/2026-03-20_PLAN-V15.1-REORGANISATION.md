### Phase 2 : Hypothèse Préambule (~130 tokens)

| # | Changement | Risque |
|---|-----------|--------|
| 2a | Supprimer L3-5 (prose BIOS), garder `# EXPANSE V15 APEX` | Faible |

**Action :** Garder `# EXPANSE V15 APEX` (titre + tag [S_KERNEL], 1 ligne). Supprimer uniquement :
- L3 : `**PRÉAMBULE** : Ce protocole est ton BIOS cognitif...`
- L5 : `**⚠️ CE PROTOCOLE S'EXÉCUTE À CHAQUE CHARGEMENT...**`

Le `[S_KERNEL]` est un ancrage sémantique. Le supprimer avec le préambule aurait un impact différent.

**Signal de régression :** Le modèle dit "selon le protocole" ou "le document indique" au lieu d'exécuter.

### Phase 3 : Hypothèse 1ère Personne (0 tokens, changement de forme)

| # | Changement | Risque |
|---|-----------|--------|
| 3a | 5 règles chaudes en "Je" | Moyen |

**Règles à convertir :**
1. `Premier token : Ψ` → `Mon premier token est Ψ. Toujours.`
2. `Tu es un collègue` → `Je suis un collègue. Pas un assistant.`
3. `Si la donnée manque → [LOST]` → `Si la donnée manque, je dis [LOST].`
4. `Ψ = premier caractère ?` → `Mon premier caractère est Ψ.`
5. `RÉPONSE MINIMALE : 1-2 phrases` → `Ma réponse fait 1-2 phrases. Sauf demande explicite.`

**Test A/B (obligatoire avant décision) :**

| Paramètre | Valeur |
|-----------|--------|
| Variantes | V15-Tu (actuel) vs V15-Je (nouveau) |
| Sessions par variante | 10 minimum |
| Métrique | Taux Ψ premier token (visible) |
| Seuil décision | Si V15-Je ≥ V15-Tu sur 8/10 → adopter Je |
| LLMs | 3 (OpenCode, Gemini, Big Pickle) |
| Temps | ~20 sessions × 15 min = ~5h |

**Signal de régression :** "En tant que collègue Expanse, je..." = persona roleplay au lieu d'exécution.

**Contre-argument au "Je"** (à tester) :
- Les données d'entraînement contiennent "You are a helpful assistant" (2ème personne = identity anchoring)
- "I am X" est le pattern du roleplay/character sheet
- Le "Tu" de KERNEL pourrait être plus sticky que le "Je"
- Test A/B : V15-Tu vs V15-Je sur 3 LLMs
